## Risks

1. M12-mini is a small sample and does not represent all real projects.
2. A larger adversarial evaluation set is required.
3. Remote prompt injection and untrusted web content cases are required.
4. Cost, latency, and token statistics need full review.
5. Flaky detection is required.
6. Context capsule and thread replacement need productization.
7. User-facing UX needs productization.
8. Install, upgrade, and permissions paths need hardening.
9. Native Mode remains experimental.
10. production_ready=false.
11. Beta and GA require larger samples and longer stability data.
